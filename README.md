# 🎵 Mitya music bot

Многофункциональный музыкальный бот для Discord с поддержкой **YouTube**, **интернет-радио** и **локальных аудиофайлов**.  
Поиск радиостанций через [radio-browser.info](https://radio-browser.info), удобная очередь, русские и английские команды.

---

## 📌 Возможности

- ▶️ Воспроизведение музыки с YouTube (поиск или прямая ссылка)
- 📻 Поиск и прослушивание интернет-радио по названию или стране
- 💾 Воспроизведение локальных MP3/аудиофайлов
- 📋 Очередь треков (добавление, пропуск, очистка)
- ⏯️ Управление: пауза, возобновление, остановка, отключение
- 🐳 Готов к запуску в Docker (docker-compose)

---

## 🚀 Установка и запуск

### Способ 1: обычный (Python)

#### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/discord-music-bot.git
cd discord-music-bot
```

#### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```


#### 3. Установка FFmpeg

Бот требует **FFmpeg** в системе.  
- **Windows**: скачайте с [ffmpeg.org](https://ffmpeg.org/download.html) и добавьте в PATH.  
- **Linux** (Debian/Ubuntu): `sudo apt install ffmpeg`  
- **macOS**: `brew install ffmpeg`

#### 4. Настройка окружения

Создайте файл `.env` в корне проекта:

```env
TOKEN=ваш_токен_бота
MUSIC_FOLDER=/путь/к/папке/с/музыкой
```

- `TOKEN` — токен Discord-бота ([получить тут](https://discord.com/developers/applications))
- `MUSIC_FOLDER` — опционально, папка с локальными треками

#### 5. Запуск

```bash
python main.py
```

---

### Способ 2: Docker (рекомендуется)

В корне проекта уже должны быть файлы:
- `Dockerfile`
- `docker-compose.yml`

#### 1. Создайте `.env` с токеном

```env
TOKEN=ваш_токен_бота
```

#### 2. Подготовьте папку для локальной музыки (опционально)

```bash
mkdir -p ./music_library
# положите туда .mp3 файлы
```

#### 3. Сборка и запуск

```bash
docker-compose up -d
```

Бот автоматически перезапустится при падении (`restart: unless-stopped`).  
Логи можно посмотреть командой `docker logs discord-music-bot`.

## 📖 Команды бота (префикс `!`)

### 🎵 YouTube / основное

| Команда | Алиасы | Описание |
|---------|--------|----------|
| `!плей <запрос или ссылка>` | `!play`, `!p` | Ищет на YouTube и добавляет в очередь |
| `!пропустить` | `!skip`, `!s`, `!скип` | Пропускает текущий трек |
| `!пауза` | `!pause` | Пауза |
| `!возобновить` | `!resume`, `!unpause` | Возобновить |
| `!стоп` | `!stop`, `!disconnect` | Очищает очередь, отключает бота |
| `!очередь` | `!queue`, `!q` | Показывает очередь (до 10 треков) |
| `!очистить` | `!clearqueue`, `!cq` | Очищает очередь |

### 📻 Радио

| Команда | Описание |
|---------|----------|
| `!радио <название или страна>` | Ищет радиостанции, показывает выбор, запускает поток |

### 💾 Локальные файлы

| Команда | Алиасы | Описание |
|---------|--------|----------|
| `!плейлокальный <имя_файла>` | `!playlocal`, `!pl` | Добавляет файл из `MUSIC_FOLDER` в очередь |
| `!список` | `!listlocal`, `!ll` | Список доступных локальных файлов (первые 20) |

## ⚙️ Конфигурация

Все настройки находятся в `config.py`. Вы можете изменить:

- `FFMPEG_STREAM_OPTIONS` — параметры FFmpeg для потоков (YouTube, радио)
- `FFMPEG_LOCAL_OPTIONS` — для локальных файлов
- `MUSIC_FOLDER` — путь к папке с музыкой (можно переопределить через `.env`)

---

## 🛠️ Примечания

- Бот автоматически подключается к голосовому каналу при вызове `!плей`, `!плейлокальный` или `!радио`
- При запуске радио **очередь очищается** – радио играет отдельно, не смешиваясь с очередью
- Для локальных файлов поддерживаются форматы, которые умеет FFmpeg (mp3, wav, ogg, flac и др.)
- Используется API `radio-browser.info` – бесплатно, без ключей

## ❓ Возможные проблемы

**Ошибка `ffmpeg not found`**  
Установите FFmpeg в систему или в Docker-образ (в `Dockerfile` уже есть).

**Бот не видит локальные файлы в Docker**  
Проверьте, что папка `./music_library` на хосте смонтирована в контейнер (см. `docker-compose.yml`). Убедитесь, что переменная `MUSIC_FOLDER` внутри контейнера указывает на `/app/music_library` (значение по умолчанию).

**Радио не играет**  
Некоторые радиостанции могут быть недоступны из-за региональных ограничений или устаревшего URL. Попробуйте другую станцию.
